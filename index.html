02 / WHY

Built on open core.
Assistants that actually work.

Noyce starts from the open Code-OSS foundation, strips out the parts that don't belong in your tools — telemetry, upsell prompts, remote-config — and adds an agent runtime that's wired into the editor, terminal, debugger and file system. Not a chat sidebar. An engineer that pairs with you on the same codebase.

03 / CAPABILITIES

Five primitives.
One agent-native workbench.

The AI layer is the centerpiece; everything else makes it useful. Each module is independent, auditable and replaceable. Run the whole stack or pick the pieces that matter for your team. No seat math, no SKU matrix — just the primitives you'd build yourself, already built.

01 / AGENTS

Multi-model, tool-using agents.

Route Claude, Gemini, Codex, and local runtimes through a single agent bus. Pick by task, cost or latency. Every agent speaks the same tool protocol: read, edit, run, flash, debug, inspect — same surface every model can use.
